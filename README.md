# Nginx Reverse Proxy Project

Данный проект демонстрирует настройку Nginx в качестве реверс-прокси для бэкенд-приложения с использованием Docker Compose.

## Структура проекта

* `nginx/nginx.conf` — основная конфигурация Nginx (оптимизация, логи, выключение лишних модулей).
* `nginx/conf.d/backend.conf` — конфигурация виртуального хоста и проксирования.
* `docker-compose.yml` — описание инфраструктуры и сетевого взаимодействия.

## Как работает схема

Схема взаимодействия компонентов реализована по принципу **Reverse Proxy**:

1.  **Клиент** отправляет HTTP-запрос на порт `80` хост-машины.
2.  **Nginx** (в контейнере `nginx_proxy`) принимает запрос.
3.  На основе директивы `proxy_pass`, Nginx перенаправляет запрос в сервис **backend** по его внутреннему имени в сети Docker.
4.  **Заголовки**: При пересылке Nginx добавляет критически важные заголовки:
    * `Host`: сохраняет оригинальное имя хоста.
    * `X-Real-IP`: передает реальный IP-адрес клиента бэкенду.
    * `X-Forwarded-For`: фиксирует цепочку прокси-серверов.
5.  **Backend** обрабатывает запрос и возвращает ответ Nginx, который передает его обратно клиенту.

## Как запустить проект

Для запуска необходимо наличие установленного [Docker](https://docs.docker.com/get-docker/) и [Docker Compose](https://docs.docker.com/compose/install/).

1. Склонируйте репозиторий (или создайте структуру файлов).
2. Перейдите в корневую директорию проекта.
3. Выполните команду для сборки и запуска контейнеров:

```bash
docker-compose up -d
```
Флаг -d запустит контейнеры в фоновом режиме.

## Как проверить результат
1. Проверка доступности
Откройте браузер или используйте curl для проверки:

```Bash
curl -I http://localhost
```
Вы должны получить HTTP-ответ от бэкенда (статус 200 OK или другой, предусмотренный вашим приложением).

2. Проверка заголовков в логах
Чтобы убедиться, что Nginx корректно передает заголовки, посмотрите логи контейнера бэкенда:

```Bash
docker-compose logs backend
```
В логах приложения вы увидите входящие запросы с заголовками X-Real-IP и X-Forwarded-For.

3. Проверка конфигурации Nginx
Вы можете проверить синтаксис конфигурации внутри запущенного контейнера:

```Bash
docker exec nginx_proxy nginx -t
```
Безопасность и оптимизация
В nginx.conf отключен вывод версии сервера (server_tokens off).

Используется upstream для удобного масштабирования бэкенда.

Дефолтные конфигурации Nginx игнорируются, используется только кастомная структура в conf.d/.
